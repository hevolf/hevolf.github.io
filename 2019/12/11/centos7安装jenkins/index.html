jenkins安装及使用（centos_7）

下载安装

  1. 官方网址：https://jenkins.io/zh/download/
  1. yum方式安装比较慢 最好直接下载rpm文件再安装

启动报错

  1. 启动报错
    service jenkins start

  1. 浏览器重启jenkins：
    http://192.168.68.10:18080/restart
    http://192.168.68.10:18080/reload

  2. 登录jenkins
    管理员初始密码

相关参数

  • /usr/lib/jenkins/：jenkins安装目录，war包会放在这里。
  • /etc/sysconfig/jenkins：jenkins配置文件，端口，JENKINS_HOME等都可以在这里配置。
  • /var/lib/jenkins/：默认的JENKINS_HOME, 默认的workspace也在此目录下。
  • /var/log/jenkins/jenkins.log：jenkins日志文件。
  • /var/lib/jenkins/secrets/initialAdminPassword：第一次登录初始密码

插件管理

  1. 镜像
    默认镜像：https://updates.jenkins.io/update-center.json
    清华镜像：https://mirrors.tuna.tsinghua.edu.cn/jenkins/updates/update-center.json

  2. 安全矩阵插件：Matrix Authorization Strategy Plugin

  3. 持续集成需要用到的工具：

    • git plugin
      This plugin integrates Git with Jenkins
    • maven integrate
      This plug-in provides, for better and for worse, a deep integration of Jenkins and Maven: Automatic triggers between projects depending on SNAPSHOTs, automated configuration of various Jenkins publishers (Junit, ...).
    • Deploy to container

构建配置

  1. 手动构建
  2. 定时构建
  3. 轮询构建
    • 定时
    • 代码有更新

构建报错

报错提示：Failed to transfer Could not transfer artifact
报错原因：maven安装后，无自定义仓库路径的操作权限导致无法pull pom依赖包（无操作权限）

自动部署

  1. 使用插件Deploy to container
    如果是在linux上部署Tomcat，需要使用/manger访问，一般需要开放访问限制Tomcat目录/webapps/manager/META-INF/context.xml

    修改Tomcat用户信息：Tomcat目录/conf/tomcat-users.xml

  2. 使用shell脚本

  1. 远程部署
    安装ssh插件
